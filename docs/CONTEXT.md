# Context — Domain Glossary

Glossary only. Implementation lives in `ARCHITECTURE.md`, `DATA.md`, ADRs.

Grow lazy — add term when first resolved in design discussion. No pre-populate.

---

## Archives

Product. Read-only Star Wars encyclopedia. Capitalized = site as product ("Star Wars Archives"), lowercase = concept of archived entries.

## Entry

Single record, single thing in Star Wars universe. Unit shown on detail page. Each entry belongs to exactly one entity type.

## Entity type

Classification of entry. v1 = six types: **Character, Film, Planet, Species, Starship, Vehicle.** Other Star Wars APIs include Droids, Locations, Organizations, Creatures — **not** entity types in v1.

## Character

Named individual in Star Wars universe — humans, aliens, droids treated as characters (e.g. C-3PO, BB-8). Replaces SWAPI "person"/"people" — "character" reads natural in Star Wars context.

## Film

Theatrical Star Wars film. v1 = 9 saga films + 2 Anthology (Rogue One, Solo). Excludes TV series, animated films (out of scope).

## Affiliation

Faction, military unit, religious order, or org a character belongs to. From akabab `affiliations[]`. Examples: "Jedi Order", "Rebel Alliance", "First Order". Distinct from **species** (biological), **homeworld** (geographic).

## Side (alignment)

Character's narrative Force side: **Light** (Jedi), **Dark** (Sith), **None** (no Force). Hand-tagged in snapshot. Drives Force-Tinted UI: hover entry tints surrounding UI accent toward that side, regardless of active global theme.

## Theme

Site global visual mode. Two values: **Jedi** (light — parchment, kyber blue), **Sith** (dark — warm near-black, crimson ember). Via `data-theme` attribute on `<html>`. Toggled via saber UI in header. Distinct from **side**: theme = user's chosen background; side = entry's narrative alignment.

## Lightsaber toggle

UI element for switching theme. Lightsaber hilt icon in header. Click ignites/extinguishes blade — ~300ms animation synced with theme transition.

## Codex

Internal name for encyclopedic surface (6 entity browsers). Not user-visible — user-facing name = "Star Wars Archives".

## Snapshot

Bundled JSON data committed to `src/shared/data/`. Generated by `scripts/build-data.ts` from upstream sources. Re-run manually when sources update — not part of `next build`.

## Repository

Class in `shared/api/` wrapping data source, exposes typed query methods per entity type (e.g. `CharacterRepository.findBySlug(slug)`). All UI and API routes read through repositories — never import snapshot JSON directly.

## Slug

URL-friendly ID for entry. Stable, human-readable, kebab-case (`luke-skywalker`, `millennium-falcon`, `a-new-hope`). Slug **is** canonical ID — no numeric IDs.
