# Context — Domain Glossary

Glossary only. Implementation details live in `ARCHITECTURE.md`, `DATA.md`, and the ADRs.

Grow this lazily — add a term the first time it's resolved in a design discussion. Don't pre-populate.

---

## Archives

The product. A read-only Star Wars encyclopedia. Capitalized when referring to the site as a product ("Star Wars Archives"), lowercase when referring to the concept of archived entries.

## Entry

A single record about a single thing in the Star Wars universe. The unit displayed on a detail page. Every entry belongs to exactly one entity type.

## Entity type

The classification of an entry. v1 covers six entity types: **Character, Film, Planet, Species, Starship, Vehicle.** Other Star Wars APIs include Droids, Locations, Organizations, Creatures — those are explicitly **not** entity types in v1.

## Character

A named individual in the Star Wars universe — humans, aliens, droids treated as characters (e.g. C-3PO, BB-8). Replaces SWAPI's term "person" / "people" because "character" reads more naturally in a Star Wars context.

## Film

A theatrical Star Wars film. v1 covers the 9 saga films + the two Anthology films Rogue One and Solo. Excludes TV series and animated films (out of scope for the dataset).

## Affiliation

A faction, military unit, religious order, or organization a character belongs to. From akabab's `affiliations[]` field. Examples: "Jedi Order", "Rebel Alliance", "First Order". Distinct from **species** (biological) and **homeworld** (geographic).

## Side (alignment)

A character's narrative side of the Force: **Light** (Jedi-aligned), **Dark** (Sith-aligned), or **None** (no Force allegiance). Hand-tagged in the data snapshot. Drives the Force-Tinted UI feature: hovering an entry tints the surrounding UI accent toward that side, regardless of the active global theme.

## Theme

The site's global visual mode. Two values: **Jedi** (light side — parchment, kyber blue) and **Sith** (dark side — warm near-black, crimson ember). Implemented via the `data-theme` attribute on `<html>`. Toggled via the saber UI in the header. Distinct from **side**: theme is the user's chosen background; side is an entry's narrative alignment.

## Lightsaber toggle

The UI element for switching theme. A lightsaber hilt icon in the header. Clicking ignites or extinguishes the blade with a ~300ms animation that runs in sync with the theme transition.

## Codex

Working internal name for the encyclopedic surface of the site (the 6 entity browsers). Not user-visible — the user-visible product name is "Star Wars Archives".

## Snapshot

The bundled JSON data committed to `src/shared/data/`. Generated by `scripts/build-data.ts` from upstream sources. Re-run manually when sources update — not part of `next build`.

## Repository

A class in `shared/api/` that wraps the data source and exposes typed query methods for one entity type (e.g. `CharacterRepository.findBySlug(slug)`). All UI and API routes read through repositories — never import the snapshot JSON directly.

## Slug

The URL-friendly ID for an entry. Stable, human-readable, kebab-case (`luke-skywalker`, `millennium-falcon`, `a-new-hope`). The slug **is** the canonical ID — no numeric IDs exist.
